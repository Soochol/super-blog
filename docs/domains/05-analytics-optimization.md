# 05. 분석 & 최적화 (Analytics & Optimization)

## 목적

트래픽, 전환, 수익 데이터를 수집하고 분석하여
키워드 전략, 콘텐츠 품질, CTA 효과를 지속적으로 개선한다.

---

## 데이터 수집

### Google Analytics 4

| 이벤트 | 트리거 | 수집 데이터 |
|--------|--------|-----------|
| page_view | 페이지 방문 | 페이지 경로, 유입 소스, 기기 |
| cta_click | 쿠팡 링크 클릭 | 제품 ID, 페이지 유형, CTA 위치 |
| filter_use | 필터/정렬 사용 | 필터 항목, 정렬 기준 |
| compare_start | 비교 페이지 진입 | 비교 제품 쌍 |
| search | 사이트 내 검색 | 검색어 |

## 분석 대시보드

### 페이지 구성 (향후 관리자 페이지)

```
[수익 요약]
  일/주/월 수익 카드
  수익 추이 차트 (라인)

[트래픽]
  방문자 추이
  유입 소스 비율 (검색/직접/소셜)
  인기 페이지 TOP 10

[전환]
  CTA 클릭률 추이
  페이지 유형별 전환율 (제품/비교/카테고리)
  제품별 클릭 순위

[SEO]
  키워드 순위 변동
  신규 인덱싱 페이지
  검색 노출 추이

[제품]
  제품별 수익 기여도
  카테고리별 성과
  가격 변동 이력
```

## 최적화 피드백 루프

### 자동 최적화

```
분석 데이터 → 자동 액션:

1. 고성과 제품 → is_featured = true → 카테고리 상단 노출
2. 저성과 페이지 → 콘텐츠 재생성 트리거
3. 클릭률 높은 CTA 변형 → 기본 설정으로 적용
4. 검색량 높은 비교 조합 → 비교 페이지 자동 생성
5. 가격 하락 감지 → /deal/ 페이지 게시
```

### 수동 최적화 (월 1회)

| 분석 항목 | 액션 |
|----------|------|
| 키워드별 순위 vs 트래픽 | 순위 높은데 트래픽 낮으면 → 제목/메타 개선 |
| 페이지별 CTR | CTR 낮은 페이지 → CTA 위치/문구 변경 |
| 카테고리별 수익 | 수익 높은 카테고리 → 제품 확충 |
| 경쟁사 분석 | 경쟁 사이트 신규 콘텐츠 모니터링 |
| 신규 카테고리 검토 | 다음 확장 카테고리 결정 |

## 리스크 및 대응

| 리스크 | 대응 |
|--------|------|
| GA4 데이터 지연 | 24~48시간 지연 감안, 실시간은 GA4 Realtime |
| 쿠팡 수익 귀속 어려움 | UTM 파라미터로 페이지별 추적 |
| 지속적 트래픽/수익 급감 | **이상 감지(Anomaly Detection) 시 즉각적인 슬랙 알림 파이프라인 구성** |
| 표본 부족 초기 | 최소 30일 데이터 후 의사결정 |
| 과최적화 | 변경 빈도 제한 (주 1회), A/B 테스트 기반 |
