# 02. AI 콘텐츠 생성 (Content Generation)

## 목적

제품 스펙 데이터를 기반으로 AI가 리뷰, 비교 분석, 구매 가이드 콘텐츠를 생성한다.
toptip.co.kr 수준의 콘텐츠 품질을 자동화로 달성하는 것이 목표.

---

## 콘텐츠 유형

### 1. 제품 리뷰 (product_review)

**트리거**: 신규 제품 등록 시 자동 생성

**입력**: 제품 스펙 + 같은 카테고리 타 제품 대비 포지셔닝 + **크롤링 기반 실제 사용자 리뷰 요약 (별점, 주요 키워드 결합)**

**출력 구조**:
```
[한 줄 요약] "고성능 게이밍에 최적화된 16인치 노트북"
[장점] 3~5개 (스펙 기반 객관적 분석)
[단점] 2~3개 (경쟁 제품 대비 약점)
[추천 대상] "이런 분에게 추천합니다"
[비추천 대상] "이런 분은 다른 제품을 고려하세요"
[스펙 하이라이트] 핵심 스펙 3~4개 강조
```

**프롬프트**:
```
당신은 IT 제품 전문 리뷰어입니다.
다음 노트북의 스펙을 분석하여 객관적인 리뷰를 작성하세요.

제품: {product_name}
스펙: {specs_json}
카테고리 평균 스펙: {category_avg_specs}
가격대: {price_range}

규칙:
- 스펙 데이터에 근거한 객관적 분석만 작성
- "직접 사용해봤다" 같은 거짓 경험담 금지
- 장단점은 같은 카테고리 타 제품과의 비교 기반
- 간결하게 작성 (500자 이내)
```

### 2. 1:1 비교 분석 (comparison)

**트리거**: 같은 서브카테고리 + 비슷한 가격대 제품 쌍 자동 생성

**비교 조합 생성 규칙**:
```
같은 서브카테고리 내에서:
- 가격 차이 30% 이내
- 같은 브랜드 내 상위/하위 모델
- 다른 브랜드 경쟁 모델
- 검색량 높은 "A vs B" 키워드 매칭
```

**출력 구조**:
```
[비교 요약] "가성비의 A vs 성능의 B"
[스펙 비교표] 주요 스펙 나란히 비교
[항목별 승자] CPU → A승, GPU → B승, 배터리 → A승...
[종합 판정] "이런 용도에는 A, 저런 용도에는 B"
[가격 비교] 현재 쿠팡 가격 기준
```

### 3. 구매 가이드 (buying_guide)

**트리거**: 수동 생성 또는 카테고리 제품 30개+ 달성 시

**출력 구조**:
```
[도입] 이 카테고리 제품을 고를 때 알아야 할 것
[선택 기준 1] CPU — 용도별 추천 (사무용 i5, 게이밍 i7+)
[선택 기준 2] GPU — 내장 vs 외장, 용도별 추천
[선택 기준 3] RAM — 8GB vs 16GB vs 32GB
...
[예산별 추천] 80만원대 / 120만원대 / 200만원대+
[FAQ] 자주 묻는 질문
```

### 4. 서브카테고리 소개 (subcategory_intro)

**트리거**: 서브카테고리 페이지용, 제품 등록 시 갱신

**출력 구조**:
```
[카테고리 설명] "2026년 게이밍 노트북 시장은..."
[트렌드] 올해의 주요 변화 (AI 프로세서, OLED 확대 등)
[추천 TOP 3] 대표 제품 3개 소개
[선택 가이드 링크] 상세 구매 가이드로 유도
```

## 콘텐츠 품질 관리

### 품질 규칙

| 규칙 | 설명 |
|------|------|
| 사실 기반 | 스펙 데이터에 근거한 분석만, 추측 금지 |
| 경험담 금지 | "직접 써봤다", "체감 성능" 같은 거짓 표현 금지 |
| 과장 금지 | "최고", "혁신적" 같은 과장 표현 금지 |
| 객관적 톤 | 전문적이고 중립적인 어조 |
| 근거 제시 | "벤치마크 기준 ~점" 처럼 수치 근거 포함 |

### 콘텐츠 갱신 정책

| 트리거 | 액션 |
|--------|------|
| 제품 스펙 변경 | 해당 제품 리뷰 재생성 |
| 가격 큰 변동 (>10%) | 관련 비교 콘텐츠 갱신 |
| 신제품 출시 | 기존 비교 콘텐츠에 새 경쟁 제품 추가 |
| 콘텐츠 90일 경과 | outdated 플래그 → 재생성 검토 |

## 비용 최적화

| 전략 | 설명 |
|------|------|
| 스펙 변경 시만 재생성 | 동일 스펙이면 기존 콘텐츠 유지 |
| prompt_hash 캐싱 | 같은 입력 → 같은 해시 → 재생성 스킵 |
| 배치 처리 | 개별 API 호출 대신 배치로 묶어 처리 |
| 가이드는 수동 트리거 | 자동 대량 생성 방지 |

**예상 비용** (제품 200개 기준):
- 초기 생성: 200 리뷰 + 100 비교 ≈ $20
- 월간 갱신: 신규 20개 + 갱신 30개 ≈ $10
- 가이드: 월 2~3개 ≈ $5
- **월 합계: ~$35**

## 리스크 및 대응

| 리스크 | 대응 |
|--------|------|
| AI 환각 (없는 스펙 생성) | 스펙 데이터만 입력, 추가 정보 생성 금지 |
| 콘텐츠 획일화 | 제품 포지션별 다른 강조점, 랜덤 문장 변형 |
| 저작권 이슈 | 스펙 데이터 기반 분석 (사실 정보는 저작권 없음) |
| SEO 중복 콘텐츠 | 비교 페이지마다 고유한 분석 관점 부여 |
